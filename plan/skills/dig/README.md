# Dig スキル

ユーザーの曖昧な要件を質問によって明確化し、整理した内容を高精度なプランとして出力する。

## 概要

このスキルは「計画立案」に特化する。実装は別のエージェントや rules/skills に委ねる設計。

**責務**:
- 要件の深掘り（AskUserQuestion による対話）
- CLAUDE.md / rules に沿った質問と計画の作成
- 計画ファイルの保存と通知

**責務外**:
- 実装の実行
- コードの生成

## トリガーフレーズ

以下のフレーズでスキルがトリガーされる:
- 「要件を整理して」
- 「プランを立てて」
- 「計画を作って」
- 「深掘りして」
- `/plan:dig`

## 計画ファイルについて

### 保存先
計画は `$HOME/.claude/plans/` に自動保存される。

### フォーマット
```yaml
---
title: [計画のタイトル]
status: pending
created: [ISO 8601 形式の日時]
---
```

### ステータス
- `pending`: 未着手
- `in_progress`: 実装中
- `completed`: 完了

## 関連 Hook

`plan-reminder` hook がセッション開始時に未完了の計画を通知する。

## 設計思想

### 単一責任
このスキルは「深掘り + 計画立案」のみに集中し、実装は別のツール/エージェントに委譲する。

### CLAUDE.md / rules 連携
プロジェクト固有のルールを読み込み、質問と計画の両方に反映させる。

### 信頼度開示
判断や提案を行う際は、信頼度をパーセンテージで開示する。
